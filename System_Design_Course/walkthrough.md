# System Design Course Walkthrough

## Overview
This course covers 40 days of System Design, from foundations to advanced case studies.
**Total Files:** 120 Markdown files.
**Location:** `G:\My Drive\Codes & Repos\System_Design_Course`

## Structure
### Phase 1: Foundations (Days 1-10)
*   **Topics:** Scalability, Networking, API Design, DB Internals, CAP, Caching, Sharding, Consensus, Transactions.
*   **Goal:** Master the basic building blocks.

### Phase 2: Building Blocks (Days 11-20)
*   **Topics:** Load Balancing, Message Queues, KV Stores, Blob Storage, Search Engines, Rate Limiting, Distributed IDs, Service Discovery, Observability.
*   **Project:** Distributed Rate Limiter.

### Phase 3: Advanced Architectures (Days 21-30)
*   **Topics:** Microservices, Event-Driven, Batch/Stream Processing, CDN, Geo-Spatial, Distributed Locking, Security, Cloud Native.
*   **Project:** Mini-KV Store (DynamoDB Clone).

### Phase 4: Case Studies (Days 31-40)
*   **Topics:** YouTube, WhatsApp, Instagram, Uber, Twitter Search, Web Crawler, Notification System, Ad Click Aggregator, Google Drive.
*   **Capstone:** Choose your own adventure (Ticketmaster/Google Maps/Crypto Exchange).

## Key Features
*   **Core Concepts:** Theoretical depth.
*   **Deep Dives:** Real-world engineering (e.g., "How Discord uses Consistent Hashing").
*   **Interview Prep:** Q&A for each topic.
*   **Code:** Python/Go snippets for algorithms (Token Bucket, Consistent Hashing, Geohash).
