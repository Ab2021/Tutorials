# Day 40: Capstone Project & Review

## 1. The Final Challenge
Choose one of the following and design it end-to-end.

### Option A: Design Ticketmaster
*   **Challenge:** High concurrency. "Taylor Swift Concert".
*   **Key:** Distributed Locking, Fair Queuing, Consistency.

### Option B: Design Google Maps
*   **Challenge:** Graph algorithms, Navigation, ETA.
*   **Key:** A* Search, Contraction Hierarchies, Geo-Spatial Index.

### Option C: Design a Crypto Exchange
*   **Challenge:** Matching Engine, Security, Speed.
*   **Key:** In-memory Matching (LMAX), Audit Logs, Hot/Cold Wallets.

## 2. System Design Framework (Recap)
1.  **Requirements:** Functional, Non-Functional, Scale.
2.  **API Design:** REST/gRPC.
3.  **Database Schema:** SQL vs NoSQL.
4.  **High Level Design:** Boxes and Arrows.
5.  **Deep Dive:** Bottlenecks, Scaling, Failure scenarios.

## 3. Final Advice
*   **Trade-offs:** There is no perfect design. Always explain "Why".
*   **Simplicity:** Simple is better than complex.
*   **Numbers:** Do the math (Back-of-the-envelope).
